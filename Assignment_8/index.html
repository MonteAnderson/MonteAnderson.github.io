  • Test results

    • After running tests between 3-key with right (d), left (a), and enter (e), it was significantly slower than the 5-key with up (w) and down (s). This makes sense, as having the ability to go up and down in a single click (like going from 7 to 4) in 1 click is much easier than having to click 4 times. After testing these two with multiple equations, the 5-key was faster in every test, as it only benefits the user to have the ability to go up and down in selection, and any area where the user would prefer to go left/right is already available in both versions, so the user has more efficient options with the 5-key version.
